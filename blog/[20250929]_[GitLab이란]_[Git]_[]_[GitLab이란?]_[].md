# 🚀 GitLab이란?

GitLab은 **Git 기반의 소스 코드 버전 관리 시스템**을 중심으로, **CI/CD(지속적 통합/배포)**, **이슈 관리**, **보안 테스트**, **배포 파이프라인**까지 통합 제공하는 **DevOps 플랫폼**입니다.
즉, **코드 작성 → 리뷰 → 테스트 → 배포 → 모니터링**을 한 곳에서 관리할 수 있게 해줍니다.

---

## 📌 GitLab의 주요 특징

### 1. **Git 저장소 관리**

* GitHub와 마찬가지로 Git 저장소를 호스팅.
* 브랜치, 태그, Merge Request(=Pull Request 유사) 기능 제공.
* 코드 리뷰, 승인 워크플로우 설정 가능.

### 2. **CI/CD (핵심 강점)**

* GitLab의 가장 큰 특징은 **CI/CD 기능이 기본 내장**되어 있다는 점입니다.
* `.gitlab-ci.yml` 파일로 빌드·테스트·배포 파이프라인 정의.
* 예: 코드 푸시 시 자동으로 테스트 실행 → 서버에 배포까지 연결.
* Kubernetes, Docker와도 쉽게 연동 가능.

### 3. **셀프 호스팅 지원**

* **GitLab Community Edition(무료, 오픈소스)** 과
  **GitLab Enterprise Edition(유료, 기업 기능 강화)** 두 가지 버전 존재.
* 기업은 GitLab을 **자체 서버**에 설치해 내부망에서 운영 가능 → 보안상 강점.

### 4. **DevOps 올인원 플랫폼**

* **Issue Tracker**: Jira처럼 이슈 관리 가능.
* **Wiki & 문서화**: 프로젝트 문서, 개발 문서 관리.
* **Merge Request 기반 리뷰**: 코드 품질 관리.
* **보안/품질 검사**: SAST(정적 분석), DAST(동적 분석) 같은 보안 테스트 기능 제공.

### 5. **권한 관리 & 보안**

* 프로젝트, 그룹, 조직 단위로 세밀한 권한 제어.
* LDAP, SSO, Active Directory 등과 통합 가능.
* 대규모 기업에서 내부 개발 환경에 맞게 커스터마이즈 용이.

---

## 📊 GitLab 아키텍처 흐름

1. **개발자**가 코드 작성 → GitLab 저장소에 Push
2. GitLab이 **CI/CD 파이프라인** 실행 (테스트, 빌드, 보안 검사 등)
3. 성공 시 자동으로 **서버 배포 / Kubernetes 클러스터 배포**
4. 결과와 로그를 GitLab에서 즉시 확인 가능

---

## 🔑 GitLab을 쓰면 좋은 경우

* 기업 내부망 환경에서 **보안과 통제**가 필요한 경우
* **CI/CD 자동화**를 적극 활용하고 싶은 경우
* **Jira, Jenkins, Confluence** 등을 따로 쓰지 않고, 한 플랫폼에서 통합 관리하고 싶은 경우
* 스타트업 → 대기업까지 확장 가능한 **올인원 DevOps 플랫폼**이 필요한 경우

---

## ⚖️ GitLab의 장단점

### ✅ 장점

* CI/CD가 내장되어 **개발–테스트–배포 자동화**가 쉬움
* **설치형 지원** → 사내망 보안 환경에서도 활용 가능
* 이슈 관리, 위키, 보안 검사까지 한 플랫폼에서 가능
* 오픈소스(CE 버전)라서 자유롭게 커스터마이즈 가능

### ❌ 단점

* UI/UX가 GitHub보다 조금 무겁고 복잡할 수 있음
* 오픈소스 커뮤니티는 GitHub만큼 크지 않음
* 서버에 직접 설치 시 리소스 소모(메모리, CPU)가 상당함
* 무료 버전(CE)에서는 일부 고급 기능 제한

---

## 🎯 정리

* GitLab은 단순한 Git 저장소가 아니라 **DevOps 통합 플랫폼**입니다.
* 오픈소스 프로젝트 → GitHub이 강세,
* **기업 내부 개발 환경 + CI/CD 자동화** → GitLab이 더 강력합니다.
